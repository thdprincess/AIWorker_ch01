04
④ 태스크 플로우(Task Flow) 작성

사용자 흐름을 시각화하다

태스크 플로우는 사용자가 특정 목표를 달성하기 위해 거치는 단계를 표준 기호로 다이어그램화한 것이다. Happy Path뿐 아니라 오류와 예외 경로도 함께 설계해야 완성도 있는 서비스가 된다.

Task Flow
태스크 플로우
특정 단일 작업의 단계와 분기를 구체적으로 표현. 가장 낮은 추상도. 개발 로직 기준으로 사용.
User Flow
유저 플로우
화면과 화면 사이의 이동 경로를 표현. 전체 내비게이션 구조를 확인하는 데 활용.
Journey Map
사용자 여정 지도
전체 경험에서의 감정·접점·채널을 포함한 거시적 관점. 리서치 단계의 산출물.

05
⑤ AI 도구(ChatGPT, Claude, Figma AI)를 활용한 설계 생산성 향상

AI로 설계를 가속하다

AI 도구는 UX 설계자를 대체하는 것이 아니라, 반복적인 작업을 자동화하고 아이디어 발산을 도와 더 중요한 판단에 집중하게 해주는 협업 파트너다.

🤖
ChatGPT
GPT-4o · OpenAI
  • 퍼소나 초안 생성 및 구체화 (나이/직업/목표/불편함)
  • HMW 질문 20개 빠르게 발산하기
  • 민원 카테고리 레이블링 대안 제안
  • 사용성 테스트 시나리오 초안 작성
프롬프트 예시:
"52세 디지털 초보 시민의 민원 서비스 퍼소나를 작성해줘. 목표, 불편함, 행동 패턴 포함."
🧠
Claude
Claude Sonnet · Anthropic
  • IA 계층 구조 논리 검토 및 개선 제안
  • 와이어프레임 주석 문서 자동 작성
  • 태스크 플로우 오류 케이스 누락 점검
  • 사용자 여정 지도 텍스트 구성
프롬프트 예시:
"공공 민원 서비스 IA에서 사용자가 자주 헤매는 구조적 문제점을 3가지 찾아줘."
🎨
Figma AI
Make Designs · Figma
  • 텍스트 설명으로 와이어프레임 자동 생성
  • 반복 컴포넌트 일괄 채우기(Fill with AI)
  • 디자인 레이아웃 자동 정렬 및 간격 조정
  • 프로토타입 인터랙션 자동 연결
프롬프트 예시:
"민원 신청 폼 화면. 이름/주민번호/주소/제출버튼 포함. 모바일 우선."

  • 2개 이상의 태스크 플로우가 작성되었는가
  • Happy Path + Error Case가 모두 포함되었는가
  • 표준 기호(원/사각/마름모)가 올바르게 사용되었는가

  • AI 생성 결과를 비판적으로 수정했는가

 10점
Chapter 1 핵심 개념 요약
CONCEPT 01
Double Diamond
발산 → 수렴을 2회 반복. 문제를 제대로 정의한 후에 해결책을 설계한다.
CONCEPT 02
카드 소팅
사용자가 정보를 직접 분류하게 하여 IA 레이블과 계층 구조를 검증한다.
CONCEPT 03
Nielsen의 5명 법칙
사용성 테스트는 5명만으로도 주요 문제의 85%를 발견할 수 있다.
CONCEPT 04
Happy Path vs Edge Case
이상적인 흐름(Happy Path)과 오류·예외 경로(Edge Case)를 모두 설계해야 한다.
CONCEPT 05
모바일 퍼스트
320px 가장 작은 화면부터 설계. 터치 영역 최소 44px. 핵심 기능 우선 배치.
CONCEPT 06
핸드오프 (Handoff)
디자이너가 개발자에게 화면 스펙, 컴포넌트, 인터랙션, 에셋을 전달하는 과정.
